多服务器部署实现下，由于多服务器下，客户端连接 id 会产生重复的情况下，
所以，必须在绑定用户Id 和 客户端连接 id 的时候加上 ip 上作为区分不同
服务器上的客户端连接id，现有的跟客户端连接id 会有冲突的地方有：

- user_id_mapping_fd
- fd_mapping_user_id
- fd_mapping_identifier
- fd_mapping_platform

以上几个要做单独的处理。另外消息转发这一块，如果发现代转发的的客户端id不是当前服务器上的
客户端id，那么需要通过 http 接口进行转发。

所以这边要处理的部分就是，保存在 redis 中的客户端连接id + ws 消息转发接口。
